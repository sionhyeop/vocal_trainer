# 보컬 트레이너 (Vocal Trainer) — 졸업작품 보고서

> 원곡 대조 음정 분석 · 발성/호흡 정량화 · LLM 기반 개인화 코칭을 하나의 파이프라인으로 통합한
> 웹 기반 보컬 트레이닝 시스템.
>
> 라이브: https://vocal-trainer-silk.vercel.app · 기술 상세: [CAPSTONE_TECHNICAL.md](./CAPSTONE_TECHNICAL.md)

---

## 1. 배경 및 목적

### 1.1 배경
"노래방 퍼펙트 스코어"로 대표되는 가창 채점은 대중적으로 친숙하지만, 기술적으로는 **반주(MR)에 내장된
가이드 멜로디(MIDI)와 마이크 피치의 단순 일치도**를 계산하는 데 머물러 있다. 이 방식은 세 가지 구조적
한계를 가진다.

1. **곡 의존성** — 가이드 데이터가 제작된 곡으로만 채점이 제한된다.
2. **단일 차원 평가** — *음정의 정오(正誤)* 만 보고, 가창의 핵심인 **호흡 운용·발성 안정성·표현**을
   설명하지 못한다.
3. **피드백 부재** — 점수만 줄 뿐 *왜 그 점수인지*, *무엇을 어떻게 고쳐야 하는지*를 알려주지 않는다.

한편 음악 정보 검색(MIR) 분야에서는 보컬 분리, 기본주파수(F0) 추정, 자동 가창 평가가 30여 년에 걸쳐 크게
발전했고, 최근에는 평가의 무게중심이 "점수 산출"에서 **"교육적 피드백"** 으로 이동하고 있다 [9]. 또한 대규모
언어모델(LLM)을 활용한 피드백이 가창 학습자의 메타인지와 실제 수행을 유의하게 향상시킨다는 연구가
보고되었다 [14]. 즉, *정답 음정을 자동으로 만들고(MIR)*, *사람의 수행을 정량화하며(신호처리)*, *그 결과를
사람의 언어로 설명하는(LLM)* 기술이 모두 성숙기에 접어들었다.

### 1.2 목적
본 작품의 목적은 위 세 흐름을 **하나의 종단(end-to-end) 시스템**으로 통합하여, "특정 곡에 한정되지 않고,
음정뿐 아니라 호흡·표현까지 분석하며, 개선 방향까지 제시하는" 보컬 트레이너를 구현하는 것이다. 세부 목표는
다음과 같다.

- **(G1) 곡 제한 해소** — 임의의 원곡에서 보컬을 분리하고 멜로디를 자동 추출해 *목표 음정(노트맵)* 을 생성.
- **(G2) 다차원 분석** — cent 단위 음정 정확도 + 호흡/발성 안정성 지표를 함께 정량화.
- **(G3) 실시간성** — 노래방처럼 부드럽고 즉각적인 실시간 피치 피드백(리본) 제공.
- **(G4) 교육적 피드백** — 약점 구간 자동 검출 + LLM 코칭으로 *학습 루프* 구성.
- **(G5) 확장 가능한 배포** — 사용자가 곡을 늘릴 수 있는 셀프호스팅 추출·배포 구조.

## 2. 시스템 아키텍처

### 2.1 설계 원칙 — "비싼 연산과 값싼 연산의 분리"
가창 분석은 성격이 다른 두 연산으로 나뉜다. **보컬 분리·F0 추출**은 GPU/CPU를 크게 쓰는 무거운 연산이지만
*1회성이고 캐시 가능*하다. 반면 **실시간 채점**은 가볍지만 *매 프레임 즉각성*이 필요하다. 본 시스템은 이
성질을 아키텍처로 환원했다: 무거운 연산은 **오프라인 배치**로 미리 수행해 결과(노트맵)만 정적 배포하고,
실시간 채점은 **사용자 브라우저**에서 수행한다. 결과적으로 상시 추론 서버가 필요 없어 운영 비용이 0에 수렴한다.

### 2.2 구성 요소
```
[원곡(YouTube)] ─다운로드/크롭─► [보컬 분리 HT-Demucs] ─► [F0/노트 추출 CREPE·pYIN·Basic Pitch]
                                                                     │  노트맵 JSON {tMs, midi}[]
   ★ 오프라인 분석 (FastAPI / 로컬·관리자 PC)                        ▼
                                                          [GitHub 리포] ─자동배포─► [Vercel CDN]
                                                                                        │ 정적 노트맵·가사
[사용자 브라우저] ── 마이크(getUserMedia) ──► [실시간 F0(MPM) + 2단 필터] ──► [cent 채점/호흡/결과/코칭]
```

- **프론트엔드**: React + Vite + TypeScript. Web Audio API, `pitchy`(McLeod Pitch Method) [3],
  Canvas 기반 노래방 리본, `soundtouchjs`(음정 시프트). zustand 상태관리.
- **분석 백엔드**: Python 3.12 + FastAPI. `yt-dlp`(음원), `ffmpeg`(크롭), Demucs(분리),
  torchcrepe/librosa/Basic Pitch(추출), 가사 프록시(lrclib). 무거운 MIR은 *오프라인*으로만 수행.
- **배포·확장**: GitHub → Vercel 자동 배포(정적 SPA + 서버리스 함수). 노트맵·고정 가사는 정적 JSON으로 CDN 분배.

### 2.3 확장 메커니즘 — 셀프호스팅 추출 큐 (차별 요소)
임의 곡을 *지속적으로* 자산화하기 위한 분산 처리 구조를 구현했다.

1. 웹 방문자가 미추출 곡의 **추출 요청** → 서버리스 함수가 요청을 **GitHub Issue(큐)** 로 적재.
2. 관리자 PC의 **로컬 워커**가 이슈를 폴링 → 로컬 백엔드로 추출 → 노트맵을 정적 경로에 커밋 → 자동 배포.
3. 웹은 노트맵 파일이 게시되면 자동으로 채점을 활성화.

이 구조는 집 PC를 인터넷에 노출하지 않고(아웃바운드 폴링만), PC가 꺼져 있어도 요청을 큐에 보존한다.
또한 **관리자 모드**(비밀 닉네임 + 서버 시크릿)로 곡·가사를 *고정(curate)* 해 품질을 관리한다.

## 3. AI 모델

본 시스템은 단일 만능 모델 대신, **단계별로 검증된 전용 모델을 조합**한다. 각 모델은 역할·근거가 분명하다.

### 3.1 보컬 분리 — Hybrid Transformer Demucs (HT-Demucs) [5]
원곡은 반주+보컬 혼합이라 곧바로 F0를 따면 베이스·신스에 오염된다. 따라서 먼저 보컬 스템을 분리한다.
HT-Demucs는 시간/스펙트럼 이중 U-Net의 내부 층을 **교차 도메인 Transformer 인코더**(도메인 내 self-
attention + 도메인 간 cross-attention)로 대체한 모델로, MUSDB-HQ 벤치마크에서 **SDR 9.0 dB**의 최첨단
분리 성능을 보인다. 본 시스템은 `--two-stems=vocals`로 보컬/반주 2분할만 수행해 연산을 절약한다.

### 3.2 멜로디 추출 — 곡 특성별 다중 F0 모델
- **CREPE** [1] — 파형을 직접 입력받는 **6-블록 CNN**으로 360-bin 피치 분포를 출력하는 데이터 기반 추정기.
  pYIN과 동등하거나 우수한 정확도. *깨끗한 단성 보컬(발라드 등)* 에 기본 사용.
- **pYIN** [2] — YIN의 임계값을 확률 분포로 두고 다중 후보를 **HMM-Viterbi** 디코딩해 안정적 피치 트랙과
  유성/무성 판정을 동시 산출. CREPE 미설치 환경의 폴백.
- **Basic Pitch** [6] — 악기 비종속 경량 신경망으로 **다성(polyphonic)·온셋·피치벤드**를 동시 추정.
  EDM·오토튠·레이어드 보컬처럼 단성 추정기가 무너지는 곡에서 프레임별 다중 피치 중 최대 진폭 음을 선택.

### 3.3 실시간 음정 — McLeod Pitch Method (MPM) [3]
사용자 음성은 브라우저에서 **MPM 기반 `pitchy`** 로 프레임마다 F0·명료도를 추정한다. MPM은 정규화 제곱차
함수(NSDF)의 피크를 골라 자기상관의 옥타브 오류에 강한 시간영역 기법으로, 프레임당 수 ms의 실시간 처리에
적합하다.

### 3.4 실시간 보정 — One Euro Filter [4] (핵심 신호처리)
원시 실시간 F0의 노이즈를 **2단**으로 제거한다. ① 중앙값 필터(window=7)로 옥타브 오검출·순간 튐 등
*소수 이상치* 를 구조적으로 제거하고, ② **1€ 필터**(속도 적응형 저역통과)로 음을 길게 끌 때는 떨림을 강하게
억제(저속·저컷오프), 빠른 도약에는 지연을 줄인다(고속·고컷오프). "느릴 땐 지터에, 빠를 땐 지연에 민감하다"는
인지 특성을 직접 반영한 알고리즘이며, 본 시스템은 `MIN_CUTOFF=1.3, BETA=0.8`로 튜닝했다.

### 3.5 개인화 코칭 — 대규모 언어모델(Claude) 🔧 *(설계 완료, 연동 예정)*
정량 분석 결과(평균/구간 cent 편차, 판정 분포, 호흡 지표, 약점 구간 Top-3)를 요약 벡터로 집계하고,
**Anthropic Claude**에 *구조화 프롬프트*(역할: 보컬 코치 / 출력 스키마: 잘한 점·핵심 약점·다음 연습 1~2개)를
주어 곡·구간에 특정된 실행 가능한 연습 처방을 생성한다. 출력은 짧고 긍정적·구체적으로 제약하고, 처방된
연습은 트레이닝 모드로 연결해 **학습 폐루프**를 구성한다 [14][15].

> 모델 선택 요약: *분리(HT-Demucs) → 추출(CREPE/pYIN/Basic Pitch) → 실시간(MPM) → 보정(1€) → 코칭(LLM)*.
> 각 단계가 그 단계에서 검증된 SOTA·표준 기법을 사용하며, 단계 간 인터페이스(노트맵·특징 벡터)로 느슨하게 결합된다.

## 4. 개발 성과

### 4.1 구현한 분석 파이프라인
- **목표 음정 자동 생성(G1 달성)**: HT-Demucs 보컬 분리 → 다중 F0 추출 → 옥타브 보정·점프 평활화 →
  `{tMs, midi}` 노트맵 직렬화·캐시까지의 전 과정을 구현. EDM/오토튠 곡까지 커버하도록 추출기 선택 로직 포함.
- **실시간 채점 엔진(G3 달성)**: 마이크 F0 → 2단 지터 보정 → 재생 타임라인 정합 → cent 환산 →
  노트 단위 판정(Perfect/Great/Good/Miss)까지를 브라우저에서 실시간 수행. 노래방식 Canvas 리본으로 시각화.
- **호흡·발성 정량화(G2 달성)**: RMS 포락선 기반 프레이즈/호흡점 분할, 발성 안정성(0~100), 최장 발성
  구간, 유성/기식 비율을 산출. *마이크 단일 채널* 로 임상 다중센서 지표의 대리(proxy)를 추정.
- **약점 구간 검출**: 타임라인 버킷별 평균 cent 편차 상위 N구간을 자동 추출, 결과 화면에서 해당 시점으로
  즉시 점프·재청취 가능.

### 4.2 핵심 기술 성과
- **2단 지터 보정(중앙값+1€ 필터)**: 단순 평활화의 *지연-떨림 트레이드오프*를 속도 적응으로 해소.
  인접 프레임 변화량을 약 23분의 1 수준(2.25→0.096 semitone, 정지음 기준)으로 안정화하면서도 빠른
  슬라이드·도약에는 즉각 반응 — 상용 노래방의 "부드럽지만 굼뜸"을 동시에 극복.
- **음향 누설(MR 유입) 차단**: 통화/Zoom식 AEC(echoCancellation)를 상시 적용해, 스피커로 재생되는
  반주가 마이크로 새어 들어와 피치를 오염시키는 문제를 구조적으로 해결.
- **셀프호스팅 추출 큐(G5 달성)**: GitHub Issue 큐 + 로컬 워커 + 자동 배포로, *상시 GPU 서버 없이*
  사용자 요청 곡을 채점 자산으로 전환하는 분산 처리 파이프라인을 구현·실증.

### 4.3 부가 기능
- **트레이닝 게임 3종(G4 보강)**: 음역대 클라이머(음역 확장), 음 듣고 맞히기(청음/상대음감),
  멜로디 따라부르기(청음+가창). 레벨·별 진행으로 반복 동기를 부여, 코칭의 연습 처방과 연결.
- **내 목소리 변조**: 가창 녹음을 `soundtouchjs`로 피치만(템포 유지) 이동해 재생, 자기 음성 모니터링.
- **개인 대시보드·기록**: 세션별 점수·약점·음역대를 로컬 저장, 홈에서 "이어서 부르기"로 재도전.
- **관리자 모드 + 데이터 파이프라인**: 곡/가사 큐레이션, 노트맵 정적 동기화 스크립트, GitHub→Vercel 자동배포.

### 4.4 기술 스택
React + Vite + TypeScript(프론트), FastAPI + Python 3.12(분석), Web Audio API · pitchy · soundtouchjs,
Demucs · torchcrepe · Basic Pitch · librosa(MIR), Vercel 서버리스 · GitHub Actions형 자동배포.

## 5. 결과

### 5.1 산출물
- **동작하는 라이브 서비스**: 정적 배포 + 서버리스 함수로 구성된 웹앱을 공개 URL로 배포
  (https://vocal-trainer-silk.vercel.app). GitHub push마다 자동 재배포되는 CI/CD 파이프라인 완비.
- **검증된 분석 파이프라인**: 다국가 인기곡 5개 장르(케이팝·제이팝·팝·발라드·트로트) 차트를 대상으로
  보컬 분리→멜로디 추출을 배치 수행, 노트맵 자동 생성을 실증(추출 진행률·캐시 포함). 추출된 곡은
  브라우저에서 즉시 실시간 채점 가능함을 확인.
- **종단 데이터 흐름 실증**: "웹 추출 요청 → GitHub 큐 적재 → 로컬 워커 처리 → 자동 커밋 → 재배포 →
  라이브 반영"의 전 과정을 실제 배포 환경에서 E2E로 검증.

### 5.2 평가 지표와 산식
음정 정확도는 노트 단위 판정에 가중치를 부여해 산출한다.
```
accuracy = (Perfect×100 + Great×80 + Good×55) / ticks      # 0~100점
```
판정 임계는 가창 평가의 통상 관행인 **반음의 1/3(≈33 cent) 이내 만점권** 과 정렬한 *관대한* 설정을 쓰며 [9],
콤보 보너스로 연속 정확 가창을 추가 보상한다. 발성 안정성은 `stability = clip(100 − meanJitter×200, 0..100)`,
호흡 지지력은 최장 발성 구간으로 정량화한다. 등급(S/A/B) 대신 **100점 + 점수대별 색·격려 문구 + 약점 구간**
으로 제시해, "직관적 점수 + 개선 지점"이라는 교육적 방향(§1.1)을 따른다.

### 5.3 의의
- 상용 노래방의 *곡 제한·단일 차원·피드백 부재* 라는 세 한계를, 각각 **MIR 자동 생성 · 다차원 분석 ·
  (LLM)교육 피드백**으로 대응하는 통합 설계를 제시·구현했다.
- 무거운 MIR과 가벼운 실시간 채점을 **시공간적으로 분리**하는 아키텍처로, 상시 추론 서버 없이도 동작하는
  *비용 효율적이고 확장 가능한* 가창 분석 서비스의 한 형태를 보였다.

### 5.4 한계와 향후 과제
- **호흡 분석의 정밀화**: 현재는 음향 단일 채널 *상대 지표* 다. 의도된 비브라토와 불안정한 떨림을 주기성
  분석으로 분리하고, 호흡 모델을 정교화할 필요가 있다(🔧).
- **LLM 코칭 연동(🔧)**: C1~C4 설계는 완료되었으며, Claude 연동·프롬프트 안전성·비용 캐싱이 다음 단계다.
- **정밀 가사 정렬(🔧)**: WhisperX 기반 단어 단위 정렬로 가사-음정 싱크를 강화.
- **추출 품질 자동 검증**: 분리·추출 실패(예: 합창·강한 이펙트) 자동 감지 및 재추출 전략.
